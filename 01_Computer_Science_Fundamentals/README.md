# Computer Science Fundamentals (計算機科學基礎)

計算機科學基礎是所有後端工程師的根基，涵蓋資料結構與演算法、計算機網路和作業系統三大核心領域。這些知識不僅是白板面試的必考範疇，更是理解系統設計、效能調優和底層原理的前提條件。

## 學習建議

1. **算法優先建立刷題習慣**：資料結構與算法是大廠面試的核心，建議每天 2-3 題 LeetCode，持續 3-6 個月，並結合後端實際應用場景（Redis 資料結構、MySQL 索引原理）加深理解。
2. **網路由上到下深度理解**：從 HTTP 應用層往下理解 TCP 可靠傳輸，再到 DNS、TLS 細節。面試高頻考點包括：TCP 握手/揮手、HTTP/1.1 vs HTTP/2 vs HTTP/3、HTTPS 握手流程。
3. **作業系統聚焦 I/O 與並發**：後端工程師最需深入理解的是 I/O 模型（epoll/select/poll）、進程 vs 線程 vs 協程的差異，以及記憶體管理機制，這些直接影響服務效能瓶頸的判斷。
4. **三個領域相互關聯**：作業系統的 I/O 多路復用（epoll）支撐了高效能網路服務；B+ 樹（資料結構）是資料庫索引的底層；一致性雜湊（算法）應用於分散式快取——學習時應有意識地建立跨領域連結。
