# 軟體開發模型 (Software Development Models)

軟體開發模型（也稱為軟體開發生命週期模型，SDLC）是軟體工程中的一個基本概念，它為軟體開發的各個階段（從需求分析到部署維護）提供了一個結構化的框架。選擇合適的開發模型對於專案的成功至關重要，它影響著專案的管理方式、團隊的協作模式、風險的應對策略以及最終產品的交付效率。

本章節涵蓋了幾種業界最重要和最常被討論的開發模型，從傳統的瀑布模型到現代敏捷開發的核心框架 Scrum。理解這些模型的原理、優缺點和適用場景，有助於在不同的專案環境下做出明智的技術和管理決策。

## 學習建議

1.  **理解演進脈絡**: 從瀑布模型的嚴格線性流程，到敏捷開發擁抱變化的迭代思想，再到 Scrum 框架的具體實踐，理解它們之間的演進關係和各自為了解決什麼問題而誕生，是掌握這些概念的關鍵。
2.  **敏捷不是銀彈**: 深入理解敏捷的核心價值觀和原則，而不是僅僅停留在 Scrum 的表面形式（如開站會）。要認識到敏捷並非適用於所有專案，在需求穩定、高可靠性要求的場景下，瀑布模型的思想仍有其價值。
3.  **Scrum 的角色、事件與產出物**: 精確掌握 Scrum 的「3-5-3」結構（3個角色、5個事件、3個產出物）。理解每個元素的目的以及它們之間如何相互作用，共同構成一個用於檢視和調適的閉環。
4.  **理論結合實踐**: 思考在真實的專案中，這些模型是如何被應用的。例如，一個大型企業的專案可能在宏觀層面採用瀑布式的里程碑管理，而在具體的開發團隊內部則採用 Scrum 進行迭代開發。這種混合模式在業界非常常見。
