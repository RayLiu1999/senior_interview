# System Design and Architecture (系統設計與架構)

系統設計是資深後端工程師面試中最關鍵的環節。本章節涵蓋了從微服務架構、設計模式到大規模系統設計案例的完整知識體系。

## 子分類索引

### 1. [Distributed Systems Theory (分散式系統理論)](./Distributed_Systems_Theory/README.md)

**核心必修**。深入探討 CAP 定理、一致性模型、共識演算法 (Raft/Paxos) 等分散式系統的基石。這是設計高可用系統的理論基礎。

### 2. [Domain-Driven Design (領域驅動設計)](./Domain_Driven_Design/README.md)

**進階必修**。學習如何透過 Bounded Context 劃分系統邊界，並運用 Aggregate、Entity 與 Value Object 構建充血模型，解決複雜業務邏輯。

### 3. [Micro Service (微服務架構)](./Micro_Service/README.md)

介紹微服務的核心概念、拆分策略、服務發現、API Gateway 以及常見的通訊模式。

### 4. [Design Patterns and Principles (設計模式與原則)](./Design_Patterns_and_Principles/README.md)

包含 SOLID 原則、常見的 GoF 設計模式以及在後端開發中的實際應用。

### 5. [Software Architecture (軟體架構)](./Software_Architecture/README.md)

探討 Clean Architecture、Hexagonal Architecture 等高階架構模式。

### 6. [Large Scale System Design Cases (大規模系統設計案例)](./Large_Scale_System_Design_Cases/README.md)

經典的系統設計面試題，如設計 URL Shortener、Rate Limiter、Chat System 等。

### 7. [Software Development Models (軟體開發模式)](./Software_Development_Models/README.md)

介紹 Agile, Scrum, Waterfall 等開發流程與方法論。

---

## 學習路徑建議

1. **打好基礎**: 先閱讀 **Distributed Systems Theory**，理解系統設計的物理限制與權衡。
2. **掌握模式**: 熟悉 **Design Patterns** 和 **Micro Service** 的常見模式。
3. **實戰演練**: 進入 **Large Scale System Design Cases**，嘗試自己設計並與範例對照。
