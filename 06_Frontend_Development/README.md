# 06. 前端開發 (Frontend Development)

前端開發是現代軟體工程中不可或缺的一環，它專注於使用者直接互動的介面 (UI) 和體驗 (UX)。一個優秀的後端系統需要一個同樣出色、高效且可維護的前端來呈現其價值。本章節涵蓋了主流前端框架、狀態管理、路由、測試以及構建工具等核心知識，旨在幫助後端工程師更好地理解「大前端」生態，並在需要時能夠進行全端開發或與前端團隊進行更高效的協作。

## 學習建議

1. **理解框架核心設計**: 無論是 Vue、React 還是 Angular，理解其核心設計哲學（例如，Vue 的響應式系統、React 的虛擬 DOM 和單向數據流）比單純記憶 API 更重要。
2. **掌握組件化思維**: 現代前端開發以組件為基礎。學習如何將複雜的 UI 拆分為可複用、可組合的組件是關鍵技能。
3. **關注狀態管理**: 隨著應用複雜度增加，狀態管理成為核心挑戰。理解何時使用本地狀態，何時需要像 Pinia 或 Redux 這樣的全域狀態管理器。
4. **熟悉構建生態**: 了解像 Vite、Webpack 這樣的現代前端構建工具如何工作，它們如何處理模組打包、代碼分割和開發伺服器，這對於專案優化和問題排查至關重要。
