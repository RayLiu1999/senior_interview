# MongoDB

MongoDB 是最流行的 NoSQL 文檔資料庫之一。作為資深後端工程師，您需要深入理解 MongoDB 的資料模型、索引機制、擴展策略以及與傳統 SQL 資料庫的差異。本章節涵蓋了面試中最常被考察的 MongoDB 核心主題。

## 學習建議

1.  **理解文檔模型**: MongoDB 的文檔導向特性是其核心優勢，要能清楚說明與關聯式資料庫的差異。
2.  **掌握索引原理**: MongoDB 的索引機制與 SQL 資料庫類似但有差異，要能根據查詢模式設計合適的索引。
3.  **熟悉聚合管線**: Aggregation Framework 是處理複雜查詢和資料分析的強大工具。
4.  **理解擴展策略**: Sharding 和 Replication 是 MongoDB 實現高可用性和水平擴展的關鍵機制。
5.  **實踐資料建模**: Embedding vs. Referencing 的選擇是 MongoDB 資料建模的核心，要能根據存取模式做出權衡。
